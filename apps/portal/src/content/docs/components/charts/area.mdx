The Area chart fills the region beneath a line using the `CartesianChart`
component with a series `type` of `CartesianSeriesVisualizationType.Area`. The
filled area emphasizes magnitude and volume over time, which makes it a good fit
for resource-utilization metrics such as CPU or memory usage.

## Time-series area

Area charts are frequently driven by time-series data using the
`{ timestamp, value }` point shape against a `ScaleType.Time` X axis. The
`units` field appends a suffix to tooltip values.

<DocsPage.ChartExample
  client:only
  code={`import {
  AxisLocation,
  AxisType,
  CartesianChart,
  CartesianSeriesVisualizationType,
  LegendPosition,
  PopoverServiceProvider,
  ScaleType,
} from "@harness/dashboard-visualization";
import "@harness/dashboard-visualization/style.css";

const series = [
  {
    name: "CPU Usage",
    color: "#3b82f6",
    type: CartesianSeriesVisualizationType.Area,
    data: [
      { timestamp: 1717200000000, value: 34.2 },
      { timestamp: 1717200060000, value: 41.8 },
      { timestamp: 1717200120000, value: 38.5 },
    ],
    units: "%",
  },
];

const xAxis = {
  type: AxisType.X,
  location: AxisLocation.Bottom,
  scale: ScaleType.Time,
  tickLabels: true,
  tickCount: 4,
  gridLines: false,
};

const yAxis = {
  type: AxisType.Y,
  location: AxisLocation.Left,
  scale: ScaleType.Linear,
  size: 56,
  tickLabels: true,
  gridLines: true,
  min: 0,
  max: 100,
  tickCount: 4,
  getLabel: (value) => value + "%",
};

render(
  <PopoverServiceProvider>
    <div style={{ width: 480, height: 280 }}>
      <CartesianChart
        series={series}
        xAxisOption={xAxis}
        yAxisOption={yAxis}
        showXAxis
        showYAxis
        legend={LegendPosition.None}
        padding={{ right: 24 }}
        downsampling={{ dataPointLimit: 120 }}
      />
    </div>
  </PopoverServiceProvider>
);`}
/>

## Area with a percentile overlay

Pair an area series with a [Dashed line](/components/charts/dashed-line) on the
same axis to show a reference threshold over the filled magnitude.

```tsx
const series = [
  {
    name: "Memory Usage",
    color: "#22c55e",
    type: CartesianSeriesVisualizationType.Area,
    data: memoryData, // { timestamp, value }[]
    units: "%",
  },
  {
    name: "90th Percentile",
    color: "#f59e0b",
    type: CartesianSeriesVisualizationType.DashedLine,
    data: memoryData.map(({ timestamp }) => ({ timestamp, value: p90 })),
    units: "%",
  },
];
```

## Usage

```tsx
import {
  AxisLocation,
  AxisType,
  CartesianChart,
  CartesianSeriesVisualizationType,
  PopoverServiceProvider,
  ScaleType,
} from "@harness/dashboard-visualization";
import "@harness/dashboard-visualization/style.css";

// An area series is a Series whose `type` is Area. It accepts either { x, y } or
// { timestamp, value } data points depending on the X axis scale.
<PopoverServiceProvider>
  <CartesianChart
    series={[
      {
        name: "series-1",
        color: "#3b82f6",
        type: CartesianSeriesVisualizationType.Area,
        data: [{ timestamp: 1717200000000, value: 34.2 }],
        units: "%",
      },
    ]}
    xAxisOption={{
      type: AxisType.X,
      location: AxisLocation.Bottom,
      scale: ScaleType.Time,
    }}
    yAxisOption={{
      type: AxisType.Y,
      location: AxisLocation.Left,
      scale: ScaleType.Linear,
      min: 0,
      max: 100,
    }}
  />
</PopoverServiceProvider>;
```

## Guidelines

<Aside type="tip">
  Anchor the Y axis at zero so the filled area accurately represents magnitude,
  and prefer a single area series — stacking many translucent fills quickly
  becomes hard to read.
</Aside>

<Aside>
  For high-resolution time series, set a `downsampling` config and use `groupId`
  to keep linked CPU/memory charts in sync.
</Aside>
