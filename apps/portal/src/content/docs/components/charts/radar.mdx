The `RadarChart` plots one or more series across several shared axes arranged
radially, producing a "spider" shape per series. Use it to compare entities
across the same set of dimensions — evaluation scores by category, capability
profiles, or multi-metric quality comparisons.

## Multi-series radar

Define the `axes` once, then provide one `series` per entity. Each series'
`data` is a list of `{ axis, value }` points whose `axis` must match an axis
`name`.

## Usage

```tsx
import {
  LegendPosition,
  PopoverServiceProvider,
  RadarChart,
} from "@harness/dashboard-visualization";
import "@harness/dashboard-visualization/style.css";

// `axes` defines the spokes; each series plots one { axis, value } point per axis.
<PopoverServiceProvider>
  <div style={{ height: 250 }}>
    <RadarChart
      axes={[{ name: "Speed" }, { name: "Reliability" }, { name: "Safety" }]}
      series={[
        {
          name: "Product A",
          color: "#ff6384",
          showPoints: true,
          data: [
            { axis: "Speed", value: 80 },
            { axis: "Reliability", value: 90 },
            { axis: "Safety", value: 85 },
          ],
        },
      ]}
      levels={5}
      legendPosition={LegendPosition.Bottom}
    />
  </div>
</PopoverServiceProvider>;
```

## Guidelines

<Aside type="tip">
  Keep every series on the same value scale (for example all `0`–`1` or all
  `0`–`100`) and ensure each series provides a point for every axis, otherwise
  the shapes are not comparable.
</Aside>

<Aside>
  Radar charts read best with three to eight axes and only a few overlaid
  series. With many series the overlapping areas become hard to distinguish — a
  grouped [Bar](/components/charts/bar) chart may be clearer.
</Aside>
