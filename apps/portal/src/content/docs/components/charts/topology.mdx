The `TopologyChart` renders an interactive node-and-edge graph driven by D3. Use
it to visualize relationships and hierarchies — organization trees, service
dependency maps, or pipeline graphs — with optional dragging, zooming, and
auto-fit layout.

<Aside type="note">
  Charts come from the `@harness/dashboard-visualization` package, which is
  published separately from `@harnessio/ui` and is not part of this portal's
  live-example runtime. The examples below are shown as code rather than
  interactive previews. In your app, import the stylesheet once
  (`import "@harness/dashboard-visualization/style.css"`).
</Aside>

## Renderers

`TopologyChart` requires a `nodeRenderer` and an `edgeRenderer`. The library
ships default renderer delegates that you register on the renderer services,
typically as module-level singletons so they are created once.

```tsx
import {
  DefaultEdgeRenderer,
  DefaultNodeRenderer,
  TopologyEdgeRendererService,
  TopologyNodeRendererService,
} from "@harness/dashboard-visualization";

const nodeRenderer = new TopologyNodeRendererService();
nodeRenderer.withDelegate(new DefaultNodeRenderer());

const edgeRenderer = new TopologyEdgeRendererService();
edgeRenderer.withDelegate(new DefaultEdgeRenderer());

export { nodeRenderer, edgeRenderer };
```

## Graph from nodes and edges

Nodes are a flat array. Each edge references its `fromNode`/`toNode` and is
pushed onto the `edges` array of both endpoints so the graph is connected.

```tsx
import {
  TopologyChart,
  type DefaultTopologyEdge,
  type DefaultTopologyNode,
} from "@harness/dashboard-visualization";
import "@harness/dashboard-visualization/style.css";
import { edgeRenderer, nodeRenderer } from "./topology-singletons";

function createNode(name: string, depth: number): DefaultTopologyNode {
  return {
    nodeType: "default-node",
    name,
    type: `Level ${depth}`,
    color: NODE_COLORS[depth % NODE_COLORS.length],
    edges: [],
  };
}

function createEdge(
  from: DefaultTopologyNode,
  to: DefaultTopologyNode,
): DefaultTopologyEdge {
  const edge: DefaultTopologyEdge = {
    edgeType: "default-edge",
    fromNode: from,
    toNode: to,
  };
  from.edges.push(edge);
  to.edges.push(edge);
  return edge;
}

const root = createNode("Organization", 0);
const engineering = createNode("Engineering", 1);
createEdge(root, engineering);
const nodes = [root, engineering];

<TopologyChart
  nodes={nodes}
  nodeRenderer={nodeRenderer}
  edgeRenderer={edgeRenderer}
  layoutType={"vertical-graph-layout" as never}
  draggableNodes={false}
  zoomable
  shouldAutoZoomToFit
/>;
```

## Usage

```tsx
import { TopologyChart } from "@harness/dashboard-visualization";
import "@harness/dashboard-visualization/style.css";
import { edgeRenderer, nodeRenderer } from "./topology-singletons";

// Provide a flat `nodes` array plus the node and edge renderer services.
<TopologyChart
  nodes={nodes}
  nodeRenderer={nodeRenderer}
  edgeRenderer={edgeRenderer}
  draggableNodes={false}
  zoomable
  shouldAutoZoomToFit
/>;
```

## Guidelines

<Aside type="tip">
  Create the node and edge renderer services once (module-level singletons)
  rather than per render, and enable `shouldAutoZoomToFit` so graphs of varying
  size always frame correctly on mount.
</Aside>

<Aside>
  Unlike the other charts, the Topology graph manages its own pointer
  interactions and does not require a `PopoverServiceProvider` wrapper. Use a
  custom `tooltipComponent` when you need rich node details.
</Aside>
