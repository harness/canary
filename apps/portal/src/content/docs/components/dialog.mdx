The `Dialog` component provides an accessible and customizable dialog overlay for displaying important content or requesting user input. It supports different sizes, themes, and can include an icon or logo in the header.

## Usage

```typescript jsx
import { Dialog } from "@harnessio/ui/components";

// ...

return (
  <Dialog.Root>
    <Dialog.TriggerPrimitive asChild>
      <Button>Open Dialog</Button>
    </Dialog.TriggerPrimitive>

    <Dialog.Content size="md">
      <Dialog.Header icon="plus-circle">
        <Dialog.Title>Dialog Title</Dialog.Title>
        <Dialog.Description>Optional description text</Dialog.Description>
      </Dialog.Header>

      <Dialog.Body>
        Dialog content goes here
      </Dialog.Body>

      <Dialog.Footer>
        <Dialog.Close>Cancel</Dialog.Close>
        <Button variant="primary">Confirm</Button>
      </Dialog.Footer>
    </Dialog.Content>
  </Dialog.Root>
)
```

## `Anatomy`

All parts of the `Dialog` component can be imported and composed as required.

```tsx
<Dialog.Root>
  <Dialog.TriggerPrimitive />
  <Dialog.Content>
    <Dialog.Header>
      <Dialog.Title />
      <Dialog.Description />
    </Dialog.Header>
    <Dialog.Body />
    <Dialog.Footer>
      <Dialog.Close />
    </Dialog.Footer>
  </Dialog.Content>
</Dialog.Root>
```

### `Description`

Optional description text below the title.

<Aside>
  The `Description` component serves two purposes: it provides visible
  descriptive text and is also used for accessibility, describing the modal to
  screen readers. If you omit it, you should use the `aria-describedby`
  attribute instead, pointing to the ID of element(s) that describe the dialog's
  purpose.
</Aside>
