The Tooltip component provides additional information for an element in a small overlay.

import { DocsPage } from "@/components/docs-page";

<DocsPage.ComponentExample
  client:only
  code={`<TooltipProvider>
    <Tooltip
        title="Full-Stack Development"
        content="A comprehensive approach to building applications that covers both front-end and back-end technologies."
    >
        <Button>Trigger</Button>
    </Tooltip>
</TooltipProvider>`}
/>

## Usage

```typescript jsx
import { Tooltip, TooltipProvider } from '@harnessio/ui/components'

// ...

return (
  <TooltipProvider> {/* typically wrapped around the entire app */}
    {/* ... */}
    <Tooltip
        title="Title"
        content={<p>This is a tooltip</p>}
    >
      <span>Hover me</span>
    </Tooltip>
  </TooltipProvider>
)
```

## `withTooltip` HOC

HOC for adding Tooltip support to any component.
For the HOC to work correctly, the component must use forwardRef.

```typescript jsx
import { withTooltip } from "@harnessio/ui/components";

// ...

const comp = withTooltip(yourComp);
```

## Variants

The `Tooltip` component supports two variants: `default` and `inverse`. The inverse variant provides a high-contrast appearance with inverted colors.

<DocsPage.ComponentExample
  client:only
  code={`
    <TooltipProvider>
      <ButtonLayout>
        <Tooltip
          title="Default Variant"
          content="This is a default tooltip with standard colors."
        >
          <Button>Default</Button>
        </Tooltip>

        <Tooltip
          variant="inverse"
          title="Inverse Variant"
          content="This is an inverse tooltip with high-contrast colors."
        >
          <Button>Inverse</Button>
        </Tooltip>
      </ButtonLayout>
    </TooltipProvider>

`}
/>

## Sides

The `Tooltip` component can be displayed to any side of the trigger element. If there isn't any room to display the tooltip on the preferred side, the tooltip will instead be displayed on the opposite side. Using the `side` prop, the `Tooltip` component supports `top`, `bottom`, `left` and `right` sides.

## Alignment

The `Tooltip` component can be aligned relative to the anchor or trigger element using the `align` prop. It supports `start`, `center`, and `end` values. This controls how the tooltip is positioned along the side it appears on.

<DocsPage.ComponentExample
  client:only
  code={`
    <TooltipProvider>
      <ButtonLayout orientation="vertical">
      <Tooltip
        title="Full-Stack Development"
        content="A comprehensive approach to building applications that covers both front-end and back-end technologies."
      >
        <Button>Center align</Button>
      </Tooltip>
      <Tooltip
        align="start"
        title="Full-Stack Development"
        content="A comprehensive approach to building applications that covers both front-end and back-end technologies."
      >
        <Button>Start align</Button>
      </Tooltip>

        align="end"

        content="A comprehensive approach to building applications that covers both front-end and back-end technologies."
      >
        <Button>End align</Button>
      </Tooltip>
      </ButtonLayout>
    </TooltipProvider>
  `}
/>
