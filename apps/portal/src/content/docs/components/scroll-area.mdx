The `ScrollArea` component offers a flexible, customizable container with scrollbars for overflowing content. Use it to display content that exceeds the visible area, supporting both horizontal and vertical scrolling.

## Usage

```typescript jsx
import { ScrollArea } from '@harnessio/ui/components';

//...

return (
  <ScrollArea className="w-[300px] h-[200px] border">
    <div className="p-4 space-y-4 w-[700px] h-[300px]">
      <p>Content that exceeds the viewport height and width.</p>
      <p>More content...</p>
      <p>Even more content...</p>
    </div>
  </ScrollArea>
);
```

### Directions

The `ScrollArea` component supports the `direction` prop to set the text direction. It accepts values like `ltr` (left-to-right) and `rtl` (right-to-left). The default value is `ltr`.

### Scroll Events

The `ScrollArea` component provides several event handlers to respond to scroll actions:

- `onScrollTop`: Triggered when the scroll area is scrolled to the top.
- `onScrollBottom`: Triggered when the scroll area is scrolled to the bottom.
- `onScrollLeft`: Triggered when the scroll area is scrolled to the left.
- `onScrollRight`: Triggered when the scroll area is scrolled to the right.

`rootMargin` and `threshold` props can be used to customize the scroll event behavior.
Use `rootMargin` to adjust the offset around the scroll area, and `threshold` to define how much of the target should be visible before triggering the scroll event.

    onScrollLeft={() => console.log('Reached left')}
    onScrollRight={() => console.log('Reached right')}
  >
    <div className="p-4 space-y-4 w-[700px] h-[300px]">

      <p>More content...</p>
      <p>Even more content...</p>

  </ScrollArea>

      Scroll Events with margin:
    </Text>

      onScrollTop={() => console.log('Reached top')}
      onScrollLeft={() => console.log('Reached left')}
      onScrollRight={() => console.log('Reached right')}
      rootMargin={{ top: '20px', right: '50px', bottom: '50px', left: '50px' }}
    >

    </ScrollArea>
  </div>

### Horizontal scroll

For correct work scroll events with horizontal scroll you can use `classNameContent="w-auto"`, basically for class name of the content, to allow it to expand based on its content width. Initially, the content will take the full width of the scroll area, but with `w-auto`, it will adjust to fit its content.
