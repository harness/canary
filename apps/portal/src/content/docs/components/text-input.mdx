The `TextInput` component is used to handle user input, providing options for styling via themes and sizes.
Also offers options for adding icons, captions, custom content, and error text for the field.

## Usage

```typescript jsx
import { TextInput } from '@harnessio/ui/components'

//...

return (
    <TextInput
    label="Success Input"
    id="someinput"
    value="input with value"
    placeholder="Enter the value"
    theme="success"
  />
)
```

## Theme

The `TextInput` component accepts a `theme` prop, which defines the visual state of the input: either `success`, `warning`, or `danger`.

When an error occurs, set `theme="danger"` and pass the error message to the `error` prop.

For warnings, set `theme="warning"` and provide the warning message via the `warning` prop.

<DocsPage.ComponentExample
  client:only
  code={`
  <Layout.Vertical gap="xl" className="min-w-[400px]">
    <TextInput
      label="Success Input"
      id="someinput"
      placeholder="Enter the value"
      theme="success"
      caption="Caption text"
    />
    <TextInput
      label="Input with Error"
      error="This field is required"
      theme="danger"
      caption="Caption text"
    />
    <TextInput
      label="Input with Warning"
      warning="Please double-check this field"
      theme="warning"
      caption="Caption text"
    />
  </Layout.Vertical>`}
/>

The `TextInput` component accepts a `size` prop, which defines the overall size of the input field.
Available options are `md` (medium) and `sm` (small)

## Prefix & suffix

The `TextInput` component supports `prefix` and `suffix` props, which allow adding content before or after the input field.

<DocsPage.ComponentExample
  client:only
  code={`
    <Layout.Vertical gap="xl" className="min-w-[400px]">
      <TextInput prefix="https://" suffix=".com" />

      <TextInput
        suffix={
          <Button variant="outline" iconOnly>
            <IconV2 name="nav-arrow-right" size="xs" />
          </Button>
        }
      />
    </Layout.Vertical>

`}
/>

## Orientation

The `TextInput` component accepts an optional `orientation` prop, which defines the layout direction of the input and its label.
Available options are `horizontal` and `vertical` (default).

<DocsPage.ComponentExample
  client:only
  code={`
  <Layout.Vertical gap="xl" className="min-w-[400px]">
    <TextInput
      label="Vertical orientation"
      error="This field is required"
      theme="danger"
      caption="Caption text"
    />
    <TextInput
      label="Horizontal orientation"
      error="This field is required"
      theme="danger"
      caption="Caption text"
      orientation="horizontal"
    />
  </Layout.Vertical>`}
/>
