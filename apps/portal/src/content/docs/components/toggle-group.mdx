The `ToggleGroup` component provides a set of toggle buttons that can be used to switch between different states or options. It is composed of subcomponents such as `ToggleGroup.Root` and `ToggleGroup.Item` to offer a structured and customizable interface.

import { DocsPage } from "@/components/docs-page";

<DocsPage.ComponentExample
  client:only
  code={`
    <ToggleGroup.Root type="multiple" value={['2']}>
      <ToggleGroup.Item value="1">Option 1</ToggleGroup.Item>
      <ToggleGroup.Item value="2">Option 2</ToggleGroup.Item>
      <ToggleGroup.Item value="3">Option 3</ToggleGroup.Item>
    </ToggleGroup.Root>
  `}
/>

## Usage

```typescript jsx
import { ToggleGroup } from '@harnessio/ui/components'

// ...

return (
  <ToggleGroup.Root
    type="multiple"
    value={['2']}
    orientation="vertical"
    variant="outline-secondary"
    onValueChange={() => {}}
>
    <ToggleGroup.Item
        value="1"
        tooltipProps={{
            title: 'Title',
            content: <p>This is a tooltip</p>
        }}
    >
        Option 1
    </ToggleGroup.Item>
    <ToggleGroup.Item value="2">Option 2</ToggleGroup.Item>
    <ToggleGroup.Item value="3">Option 3</ToggleGroup.Item>
  </ToggleGroup.Root>
)
```

## Anatomy

All parts of the `ToggleGroup` component can be imported and composed as required.

```typescript jsx
<ToggleGroup.Root>
  <ToggleGroup.Item />
</ToggleGroup.Root>
```

## Controlled and Uncontrolled Usage

The `ToggleGroup.Root` component can be used as a controlled or uncontrolled component.

- **Controlled**: Pass both the `value` prop and the `onValueChange` handler to fully control the selected value from outside the component.
- **Uncontrolled with default value**: Pass the `value` prop to set an initial selection. The component will manage its own state internally.
- **Fully uncontrolled**: Do not pass `value`. The component will manage its state entirely on its own.

## Unselectable

The `ToggleGroup.Root` component accepts a `unselectable` prop to control whether the last selected item can be deselected. When set to `true`, the component prevents users from unselecting the last remaining active toggle.

<DocsPage.ComponentExample
  client:only
  code={`
    <ToggleGroup.Root type="multiple" value={['2']} unselectable>
      <ToggleGroup.Item value="1">Option 1</ToggleGroup.Item>
      <ToggleGroup.Item value="2">Option 2</ToggleGroup.Item>
      <ToggleGroup.Item value="3">Option 3</ToggleGroup.Item>
    </ToggleGroup.Root>
  `}
/>

## Icon only

The `ToggleGroup.Item` component accepts a `iconOnly` prop. When this prop is set, the `ToggleGroup.Item` is styled to fit the dimensions of an icon, making it suitable for icon-only buttons.

<DocsPage.ComponentExample
  client:only
  code={`
    <ToggleGroup.Root type="multiple" value={['2']}>
      <ToggleGroup.Item value="1" iconOnly>
        <IconV2 name="plus" />
      </ToggleGroup.Item>
      <ToggleGroup.Item value="2" iconOnly>
        <IconV2 name="minus" />
      </ToggleGroup.Item>
      <ToggleGroup.Item value="3" iconOnly>
        <IconV2 name="xmark" />
      </ToggleGroup.Item>
    </ToggleGroup.Root>
  `}
/>

## With tooltip

The `ToggleGroup.Item` component accepts a `tooltipProps` prop. When provided, the `ToggleGroup.Item` will be wrapped in a `Tooltip` component. The `tooltipProps` object can include the following properties from `TooltipProps`: `title`, `content`, `side`, and `align`.

<DocsPage.ComponentExample
  client:only
  code={`
    <ToggleGroup.Root type="multiple" value={['2']}>

        iconOnly
        tooltipProps={{
            title: 'Title',

        }}
