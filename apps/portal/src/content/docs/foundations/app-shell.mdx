The **app shell** is the persistent chrome around product pages: navigation rail, AI chat panel, gutters, primary content, and optional canvas regions (graphs, split panes). Surface colors are defined as **component tokens** under `comp.app-shell.*` so every theme variant (24 modes) resolves the same layer roles to the correct semantic backgrounds.

Use this page for **which layer gets which class**. For elevation on nav and chat panels, see [Shadows — App shell rail shadows](/foundations/shadows#app-shell-rail-shadows). For the `Sidebar` component API, see [Sidebar](/components/navigation/sidebar).

## Background IA

The shell stacks five surface roles. Gutters and outer wrappers use the **shell** surface; elevated columns use **nav** and **chat**; page chrome uses **main content**; deep inset areas use **canvas**.

```text
┌─────────────────────────────────────────────────────────────┐
│  shell (gutters / outer wrapper)                            │
│  ┌──────┬──────────┬────────────────────────┬──────────────┐ │
│  │ nav  │  chat    │   main content         │  (optional)  │ │
│  │      │          │                        │              │ │
│  │      │          │   ┌──────────────────┐   │              │ │
│  │      │          │   │ canvas (split)   │   │              │ │
│  │      │          │   └──────────────────┘   │              │ │
│  └──────┴──────────┴────────────────────────┴──────────────┘ │
└─────────────────────────────────────────────────────────────┘
```

## Surface tokens and platformUI utility classes

Design tokens (`comp.app-shell.*`) ship in `@harnessio/ui` as `--cn-comp-app-shell-*` CSS variables in every theme. **Utility classes** (`cn-app-shell-bg`, `cn-nav-surface-bg`, etc.) are defined in **platformUI** (`apps/kitchen-sink/src/App.css`) — not in the design system package.

Attach these class names to the DOM node that owns each region's background. Each class sets `background-color` from the matching token:

## Unified navigation rail

The nav column paints on the **nav surface** wrapper. Inner `Sidebar` panels stay transparent so one surface shows through the full rail width. In platformUI, transparency rules live in `App.css`:

- `.cn-sidebar-container .cn-sidebar` — `background-color: transparent`
- `.cn-sidebar-nav-panel` — transparent, no corner radius on the panel shell

Apply `cn-nav-surface-bg` on the nav column container (not on individual sidebar items). Pair with `shadow-cn-nav-rail` on the same shell element when elevation is required.

## Themed nav and chat

For brand- or product-themed shells, add `cn-app-shell-themed` on the app shell root. Nav and chat surfaces can then override via optional theme hooks:

- `--cn-comp-app-shell-nav-theme`
- `--cn-comp-app-shell-chat-theme`

Main content and canvas **keep** the default surface tokens — only nav and chat participate in themed mode.

```html
<div
  class="cn-app-shell-bg cn-app-shell-themed"
  style="--cn-comp-app-shell-nav-theme: var(--cn-set-brand-primary-bg);"
>
  <!-- nav + chat pick up theme; main + canvas unchanged -->
</div>
```

## Implementation checklist

1. **Shell wrapper** — `cn-app-shell-bg` on the outer layout (e.g. sidebar wrapper + gutters).
2. **Nav column** — `cn-nav-surface-bg` + `shadow-cn-nav-rail` on the rail shell.
3. **Chat panel** — `cn-chat-surface-bg` + `shadow-cn-chat-rail` or `shadow-cn-chat-rail-mirror` based on chat position.
4. **Main** — `cn-main-content-surface-bg` on the primary content region.
5. **Canvas** — `cn-canvas-surface-bg` on split/graph areas only.

<Aside type="caution">
  Do not apply rail shadows to seam overlays, resize handles, or gutter strips —
  shadows belong on the nav/chat column shells only.
</Aside>
